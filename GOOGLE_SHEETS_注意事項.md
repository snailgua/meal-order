# Google Sheets 操作注意事項

本文件說明哪些 Google Sheets 操作會影響 Web App 運作，哪些不會。

---

## 不會影響 Web App 的操作（可以放心做）

- **儲存格格式** — 背景顏色、粗體、斜體、字型大小、文字顏色
- **框線** — 加底線、外框、格線
- **欄寬 / 列高** — 調整寬度高度
- **凍結窗格** — 凍結標題列方便瀏覽
- **條件格式化** — 例如金額超過 100 變紅色
- **新增其他工作表** — 加新的分頁做統計或分析都不影響
- **篩選器檢視** — 使用「資料 → 建立篩選器檢視」來排序或篩選（只影響你自己的視覺，不改動實際資料）
- **改標題列文字** — 程式不看標題文字，只看欄位位置，所以改標題文字不影響

---

## 會影響 Web App 的操作（不要做）

| 操作 | 後果 |
|---|---|
| 改變欄位順序（例如把姓名和品項對調） | 程式用固定的欄位索引讀寫，會把資料寫到錯的欄位 |
| 插入或刪除欄 | 同上，索引全部移位，資料會錯亂 |
| 改工作表名稱（例如把「訂單明細表」改名） | 程式用名稱查詢工作表，改名後會找不到 |
| 刪除工作表 | 程式找不到工作表，直接報錯 |
| 手動刪除或插入資料列 | 可能導致 rowIndex 對不上，編輯或刪除操作會動到錯的那筆 |
| 直接排序資料（非篩選器檢視） | 資料列順序改變，rowIndex 全部錯亂 |
| 手動修改狀態欄的值（如「付款人是否標記已付」） | 會影響付款追蹤邏輯；如果清楚自己在做什麼可以改，但要小心 |

## 目前的工作表結構（請勿變動）

### 訂餐場次表
場次ID, 日期, 標題, 負責人姓名, 負責人銀行名稱, 負責人銀行帳號, QR Code圖片連結, 轉帳連結, 狀態, 建立時間, 菜單圖片連結

### 訂單明細表
場次ID, 日期, 標題, 負責人姓名, 姓名, 品項名稱, 價格, 備註, 建立時間, 最後修改時間

### 付款追蹤表
場次ID, 日期, 標題, 付款人姓名, 收款人姓名, 金額, 品項名稱, 備註, 付款人是否標記已付, 收款人是否確認收到, 核銷時間

---

## 簡單原則

**格式隨便改，結構不能動。**

想在 Google Sheets 裡做排序或篩選分析，請用「篩選器檢視」（資料 → 建立篩選器檢視），不要直接排序資料。
